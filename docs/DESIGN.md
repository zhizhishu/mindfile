# DESIGN — 设计与调研综合

## 要解决的问题

AI 编码 agent 用得越久，记忆越多。三个具体病：

1. **索引撑爆预算**：每会话"自动加载"的记忆索引随条数线性增长，最终超出上下文预算（Claude Code 的 memory 索引有 ~200 行 / 25KB 的读取上限）。
2. **蒸馏不够用**：把旧记忆"蒸馏成稳定事实"是标准做法，但**大量情节史根本蒸不出事实**（"某日做了 X" 这类流水），只能干堆。
3. **时间偏见**：靠"多久没碰"判冷热，会把"一年只碰一次但绝不能丢"的铁律（账号被封、某主机 10 秒掐连接）误判成低频垃圾淡出。

## 设计原则（与被否决的替代方案）

### 渐进披露，而非全量注入
只让极小的索引层常驻，正文按需读。⚠️ 注意 `@import` 一个文件**不省预算**——被 import 的文件启动照样全量进上下文；只有"启动不加载、用工具按需读"才真省。

### MOC 两级索引，而非"按新近度分冷热"
- **热/冷二分**：把冷条移到不自动加载的 archive。有效，但顶层索引仍随总条数线性涨，且要人肉判断哪些冷。
- **MOC (采用)**：顶层只是一张"类目 → 子索引文件"的地图；②③ 全在子索引里、按需读。**永远加载层大小 = O(类目数)，与总条数脱钩**。不相关类目根本不加载（比"冷档按需读"更省——冷档整个文件都不用碰）。

### 反思固化，而非硬蒸馏
承认多数情节蒸不成事实。反思 = 把一批老情节**压成一段更高层的叙事**（几何压缩），而非逐条榨事实（线性、且有大量榨不出的残渣）。

### 耐久类型驱动保留
四类：`axiom`(铁律，永不归档) / `pattern`(可复用技法) / `workaround`(绑工具版本、会过气) / `project-state`(会腐烂)。让保留由**内在耐久度**驱动，而非 mtime。可选叠加"被 recall 命中即自动升热"，但**不做"低频自动遗忘"**——对精选事实库，自动遗忘是危险的。

**实现现状(2026-07-18 更新)**：四类耐久度**均已驱动冷打分**——`axiom` 锁热(永不列归档候选)、`workaround` / `project-state` 加冷权重(更易归档)、`pattern` 加热权重(降冷分、更抗归档)。此前只有 axiom/workaround 生效的 2/4 缺口已补齐。同期 `valid-as-of` 时态失效由 `automemory_audit.py` 半自动化(超期自动标 `needs-review`/`stale-valid-as-of`，只报不删)。

### 文件系统 + grep，而非向量库
对百来条的精选库，`grep`/结构化检索比嵌入更省更准（[Letta 实测](https://www.letta.com/blog/benchmarking-ai-agent-memory) filesystem 74.0% > mem0 向量 68.5%），且明文可审、可 diff、可版本控制。真到几百上千条且语义模糊时再考虑混合检索不迟。

## 为什么分两层、且不"全 mindfile 化"

- **项目记忆(mindfile)** 是**每项目一套分层文件**，有硬边界（禁跨项目/父级写）——适合"这个项目怎么回事"。
- **全局记忆(auto-memory)** 是**一个扁平原子 fact 池 + MOC 索引**——适合"跨项目通用教训"。

刻意**不把全局仓也拆成一堆项目文件**：扁平原子池对"通用经验"这个用途，比每项目文件爆炸更合适。两层各取所需，别互相套。

## 调研出处：借了什么、拒了什么

| 来源 | 借鉴 | 
|---|---|
| Anthropic Agent Skills / Claude Code memory | progressive disclosure 三层加载；写后量行数/字节的预算 hook |
| MemGPT / Letta | OS 式分层分页(core vs archival)；本系统的热/冷 + MOC 是其手动版(百来条无需全自动分页) |
| Stanford Generative Agents | reflection：一批情节 → 更高层洞察 |
| Obsidian MOC / LYT | 两级/多级"索引的索引"，顶层随领域数而非笔记数增长 |
| Letta Filesystem | 文件名+描述即索引；grep/语义检索兜底，不必维护全量指针 |
| Zep / Graphiti | 时态失效：会变的事实标 superseded + 带日期，而非留着误导 |
| mem0 | 写入前先查近似条、有则 update 不新增，从源头压膨胀 |

**明确拒绝**：① 为百来条上向量库(过度工程)；② 隐藏二进制自动记忆(Windsurf 式——不可审、混入过期假设、越用越错)；③ 把全局仓整份 mindfile 化(每项目文件爆炸不适合通用经验池)。

## 安全

所有会改记忆的操作：**dry-run 默认 → `--apply` 才写且先备份 → 绝不删 fact/正文 → 移动即高风险、需人工确认**。审计工具**只报不改**。重构/压缩保证**索引守恒 + 零信息损失**（`recall -q` 搜全部 fact 全文，归档来源照样命中）。
